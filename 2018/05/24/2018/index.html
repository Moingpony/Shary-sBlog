2018,第一篇自己的博客

发表于 2018-05-24
字数统计: 3.5k | 阅读时长 ≈ 4 mins.

  • 博客篇

    开篇讲讲这个博客：

    最早接触到技术博客，可能是来源于Randy Lu的技术博客。从2015年到2018，经历了博客的迁移。

    坚持做独立博客很难，坚持写博客更难。

    从决定开始学习搭建自己的博客到建成，也不过1天的时间。假如三年前直接决定搭建一个个人博客，也许也没有那么复杂。所以一旦喜欢做的事要积极去准备，尽早去做。

    为什么要写独立博客？
    现在身在一所普通二本院校，计算机专业，已经大三末尾，但感觉自己现在还是什么都不会。大一整天忙于各种事情，自己又是个懒惰的人，学过很多语言，工具，框架，但都是浅尝辄止，而又没有更多的时间来学习。 到现在终于有一点点的时间了，重新审视一下自己，专业知识学的并不怎么样，项目经验没有，东西学的嘈杂，而又不会善始善终，开始喜欢上Python，但又无从下手。

    可能是因为一些事更加坚定了我写博客想法，并且尽量写一些高质量的博客。
    刚开始是在做某些东西或者当自己做的项目出错时，去网上搜解决方案，搜到一些人的博客，先是CSDN，博客园的，再到github,stackoverflow，V2EX.然后都引流到他们的个人博客，在他们的个人博客里，他们记录了他们曾经走过的路，是如何考研，做项目，学习算法，如何读书，如何思考，如何去准备面试，如何进入谷歌，facebook，微软这样的公司实习或工作。
    有初中就开始的做的，也有大三才开始的，但他们对此有相同的态度，通过坚持，都有了自己的收获。
    他们曾经都有过像我此刻的迷茫，但他们最终都靠自己的努力实现了逆袭。
    有幸在我迷茫未来，苦苦寻求的时候，读到了他们这些文字。让我看到了方向，我也想建立一个自己的博客，我也要通过努力来实现自己的梦想，不再日复一日地去浪费时间。

  • 经历篇

  • 2015

  • 2015年，大一入学。开始接触C，C++两门语言，算是给自己的编程之路打了个基础。最初接触代码，有些好奇，也有些兴趣。再往后经历几个月，甚至是往后再经历几门语言以后，觉得难道各类语言只是简单的用一些学到的语法去解几个数学题，进行一些简单计算。连代码都变得越来越乏味，甚至觉得学这些东西根本无用。

  • 2016

  • 2016年，进入大二，进一步学习了java，php,js。各种数据库，oracle，mysql,SQL Server。开始接触一些小的完整的项目。参加过一些软件设计的比赛，最终不了了之。不过还好，这一年，接触到了Python。并且疯狂爱上了这门语言。强大的库支持，优美的结构，简洁的代码，更让我这么一个懒散的人着迷。

  • 2017

  • 2017年，这一年步入大三。接触到了更多的技术。如版本控制的Git、SVN，node.js,Ruby语言、Go语言、R语言。Python也逐步进阶，从做爬虫开始转向Python全栈开发。这一年一直在思考自己要做哪一行。迷茫的一年。

  • 2018

  • 2018年才拿出了自己的四级证。转眼大三面临结束。与惠普合作的企业课逐渐植入。与企业老师沟通，更多的是告知大数据与人工智能逐渐兴起并占据一席之地。而我又重新在java与python之间陷入了迷茫。

    直到写完这篇博客，我还是不想放弃我的Python。所以还是决定搏一搏吧。

  • 一些忠告

引用Randy的一段话回答一下刚开始的迷惑。

“但是当你开始接触（我所在的）大学的计算机相关课程，你会发现，所谓的编程学习，无非是老师教一章内容，要求你用学到的语法去解一些数学题。于是很多人选择放弃，认为编程乏味无趣，用很长的课时学习，却只能写出简单的四则运算，或者解数学题。

然而这并非编程的全部，因为这恰恰能生动地表明编程语言只不过是一种工具，你可以用它来解数学题，也能做其它更有趣、更实用的程序。

在（我所在的）大学，计算机相关课程教 C/C++。有一次，一位大三的朋友问我说，『我学 C 学了一段时间，为什么我还是感觉很没用？』我是这样回答他的：

你学 C 学了一段时间感觉没有用处并非你个人的问题，我无法跟你解释为什么，不过我可以告诉你的是，Linus Torvalds 用 C 写了 Linux 和 Git; Antirez 用 C 写了数据库 Redis. 你可能连 Git 是什么都不知道，如何用 C 写出这种水平的程序呢？

所以，当你学习 C/C++ 基础后进入困惑期时，不要因为用它写不出实用的程序而止步不前，而应该开始接触其它的编程语言。当你所学和所做的东西越来越多的时候，你慢慢就会发现，编程语言只不过是一种工具，重要的在于你想做什么，用什么语言合适去做（华软的校内游戏直播网站 SISE Game 就是例子，网站的后端用 Ruby on Rails 写，但实时聊天是用 Node.js (JavaScript) ，因为相对于 Ruby, Node.js 更加适合处理 WebSocket）。 你之所以感觉 C/C++ 没用，是因为你暂时还没有足够的眼界和能力去使用他去进行创造。

应该去学什么？我的建议是一门严谨的强类型语言（如 Java）和一门愉快的脚本语言（如 JavaScript, Ruby, Python）。当然了，如果你想学一些能立竿见影的技能，你也可以写写前端（HTML, CSS）。”

多访问一些技术论坛，你会发现外面的世界更精彩。

的确是这样，你不要指望你的学校，你的老师能教给你什么，更多的还是靠你自己。

  • 最后

第一篇博客结束了，希望各位能在有限的时间里，开发自己的兴趣，有喜欢的事情积极去做。希望各位能在自己喜欢的路上走下去，也希望各位有兴趣的不要因为目前的迷茫放弃自己的IT之路。
